# GS_Smart_car

> 智能车项目合集：用于整理智能车相关代码、调试记录、控制算法和底层驱动实践。

## 仓库定位

本仓库用于集中管理智能车相关项目内容，适合作为比赛项目、课程实践和嵌入式控制经验的展示入口。

| 项目 | 说明 |
|------|------|
| 方向 | 智能车、传感器融合、底层控制、运动控制 |
| 代码类型 | MCU 固件、驱动模块、控制逻辑、调试脚本 |
| 重点能力 | 实时控制、传感器采集、状态机、通信与调参 |
| 推荐用途 | 项目复盘、面试展示、后续智能车项目复用 |

## 建议目录结构

```text
GS_Smart_car/
├── docs/                 # 设计文档、接线说明、调试记录
├── firmware/             # MCU 固件工程
├── hardware/             # 原理图、PCB、BOM 或接线资料
├── tools/                # 上位机、调参、数据记录工具
├── data/                 # 采样数据、测试数据
└── README.md
```

## 阅读路线

1. 先看 `docs/` 中的硬件连接和系统框图；
2. 再看 `firmware/` 中的主控工程；
3. 重点关注传感器采集、控制周期、通信链路和故障处理；
4. 最后结合 `tools/` 或 `data/` 复盘调参过程。
